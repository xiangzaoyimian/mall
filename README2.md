# 裤子商城系统 (Pants Mall)

## 系统简介

裤子商城系统是一个专门针对裤子商品的全功能电商平台，包含完整的前端购物体验和后端管理功能。系统采用前后端分离架构，提供了丰富的商品展示、搜索、购物车、订单管理、用户中心等功能，同时集成了AI推荐系统，为用户提供个性化的购物体验。

## 系统架构

### 技术栈

#### 前端
- React 18
- TypeScript
- Vite 构建工具
- CSS 模块化样式
- Axios 网络请求

#### 后端
- Spring Boot 3
- MyBatis Plus 数据访问框架
- MySQL 数据库
- JWT 认证
- Spring Security 安全框架

### 系统分层

#### 前端架构
- **页面层**：包含所有React组件，负责用户界面展示和交互
- **API层**：封装与后端的通信逻辑，处理网络请求和响应
- **工具层**：提供通用工具函数，如JWT处理、地区数据等
- **样式层**：模块化CSS样式，确保界面美观一致

#### 后端架构
- **控制层**：处理HTTP请求，参数验证和响应
- **服务层**：实现业务逻辑
- **数据访问层**：与数据库交互
- **实体层**：定义数据模型
- **公共层**：提供通用工具和配置

## 系统部署

### 前端部署
1. 安装依赖：`npm install`
2. 构建项目：`npm run build`
3. 部署生成的`dist`目录到静态文件服务器

### 后端部署
1. 编译项目：`mvn clean package`
2. 运行生成的jar文件：`java -jar mall-0.0.1-SNAPSHOT.jar`
3. 数据库初始化：执行`mall.sql`脚本

### 环境要求
- 前端：Node.js 16+
- 后端：JDK 17+, MySQL 8+
- 服务器：至少2GB内存

## 系统配置

### 前端配置
- API基础URL：修改`src/api/client.ts`中的`baseURL`
- 图片URL解析：修改`App.tsx`中的`resolveImageUrl`函数

### 后端配置
- 数据库连接：修改`application.yml`中的数据库配置
- JWT密钥：修改`application.yml`中的JWT配置
- 端口设置：修改`application.yml`中的服务器端口

## 系统维护

### 日志管理
- 后端日志：`logs`目录
- 前端日志：浏览器控制台

### 常见问题
1. **登录失败**：检查账号密码是否正确，或联系管理员重置密码
2. **商品无法添加到购物车**：检查商品是否有库存
3. **订单创建失败**：检查地址信息是否完整
4. **图片无法显示**：检查图片URL是否正确
